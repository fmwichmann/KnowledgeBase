## Kontext

Die Wissensbasis liegt auf dem PC des Nutzers und wird per Git versioniert. Der Bot läuft auf einem Server. Neue oder geänderte Dokumente müssen in ChromaDB indexiert werden, ohne dass ein manueller Eingriff auf dem Server nötig ist.

## Alternativen

**Neuindexierung beim Start:** Funktioniert nur bei Serverneustart — für einen dauerhaft laufenden Server nicht geeignet.

**Manueller `/reindex`-Befehl:** Einfach, aber erfordert aktives Eingreifen nach jedem Push.

**Dateiwächter + GitHub Webhook:** GitHub benachrichtigt den Bot per HTTP nach jedem Push. Bot führt `git pull` aus. Dateiwächter erkennt Änderungen und indexiert automatisch nach.

## Entscheidung

Dateiwächter (`watchdog`) überwacht das lokale Verzeichnis der Wissensbasis auf dem Server. Ein GitHub Webhook-Endpoint (`/github-webhook`) in Flask empfängt Push-Events, führt `git pull` aus und löst die Neuindexierung aus. Der Dateiwächter dient als zusätzliche Sicherung bei direkten Dateiänderungen auf dem Server.

## Konsequenzen

**Positiv:** Vollautomatisch — Push auf GitHub genügt, Bot indexiert ohne weiteres Zutun.

**Negativ:** GitHub Webhook erfordert eine öffentliche URL (ngrok lokal, feste URL auf Server). Webhook-Secret sollte zur Absicherung gesetzt werden.

## Kontext

Der Telegram Bot soll Antworten in einem definierten Stil liefern. Die Stil-Anweisung muss bei jedem API-Call als System Prompt mitgesendet werden, da die Claude API zustandslos ist.

## Alternativen

**Hardcoded im Code:** Stil-Anweisung direkt im Python-Code. Einfachste Implementierung, aber Änderungen erfordern einen Code-Eingriff.

**Konfigurationsdatei:** Stil-Anweisung in `config/system-prompt.md`, wird beim Start geladen und bei jedem API-Call mitgesendet. Änderbar ohne Code-Eingriff, versionierbar.

**Dynamisch aus Wissensbasis:** Stil-Anweisung wird zur Laufzeit aus einem Dokument abgerufen. Maximale Flexibilität, aber für den aktuellen Stand überdimensioniert.

## Entscheidung

System Prompt als Konfigurationsdatei: `config/system-prompt.md`. Die Datei wird beim Start der Anwendung geladen und bei jedem API-Call als `system`-Rolle übergeben.

## Konsequenzen

**Positiv:** Stil ohne Code-Änderung anpassbar. Datei ist versioniert und nachvollziehbar. Konsistent mit dem dateibasierten Ansatz der Wissensbasis.

**Negativ:** Datei muss beim Deployment verfügbar sein.
